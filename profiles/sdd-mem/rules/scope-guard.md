# Guardia de Scope

Hacer SOLO lo que se pidio:

- Si se detecta algo mejorable fuera de scope, guardarlo con `/learn` o `/gotcha` y seguir
- No agregar error handling especulativo
- No mejorar codigo que funciona si no se pidio
- No refactorear de paso
- Sugerencias en UNA linea. Si el usuario quiere mas detalle, lo pide.
- Clasificar intent en cada turno: research, implementacion, investigacion, fix, open-ended
- No arrastrar modo implementacion de turnos anteriores
